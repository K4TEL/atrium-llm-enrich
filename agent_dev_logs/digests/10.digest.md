## 🎯 Goal
Turn input **PDFs (curve-only / scanned / digital-born)** and **DOCX** into the repo's LLM-friendly
targets — since the 2026-07-21 comments, specifically into **visually-rich Markdown**: page borders
and layout cues recorded as HTML comments inside page-sectioned `.md`, so the LLM keeps layout
signal a plain-text dump would lose.

## 🛠️ What landed (`456ebb2`)
- **`api_util/layout_md.py`** — dependency-free **cue vocabulary**: pure emitters returning the
  taxonomy's exact strings + a `CUE_SCHEMA` catalogue ("the list of all possible visual layout
  pieces", single-sourced and testable).
- **`api_util/docx_to_md.py`** (python-docx, MIT) — styles→headings, emphasis + explicit FONT/STYLE
  cues, alignment, tables, headers/footers + margins, page breaks → `## Page N`, images, footnotes.
- **`api_util/pdf_to_md.py`** (pdfplumber, MIT) — per-page `DOC_META`, block `BBOX`+`FONT` (blocks
  split on gaps *and* font-size change), tables, **decode-sanity check** → text-less/garbled pages
  emit `NEEDS_OCR` instead of fabricated text.
- **`api_util/doc_to_visual_md.py`** — dispatcher + CLI: pre-convert, land the `.md` in `INPUT_DIR`,
  run the pipeline unchanged (HTML-comment cues pass through as inert text).
- Optional `requirements_docmd.txt`; 4 hermetic test modules.

## 🛠️ What landed (plan §7, 2026-07-22 — second pass)
- **Unified schema:** `xml_to_md.py --format layout` renders TEITOK/ALTO to the *same* annotated MD
  (DOC_META/BBOX/PAGE_BREAK/figure cues) from their coordinates — annotated MD is now the one diet
  across PDF, DOCX, and TEITOK/ALTO. Default `markdown`/`text` output byte-identical.
- **OCR (opt-in):** `pdf_to_md.py --ocr` renders text-less pages (pypdfium2) and transcribes them
  with Tesseract `ces` (`pytesseract`), tagged `<!-- OCR: engine=tesseract, lang=ces -->`; graceful
  fallback to `NEEDS_OCR` when the binary is absent. Default behaviour unchanged.
- **Citations:** the document schema returns an optional `page`; `run_document_level` surfaces it →
  `[Source: <doc_id>, Page N]` (issue #11 §3.3).
- **Auto-convert:** `llm_client_shared.prepare_document_input()` converts `.pdf`/`.docx` to a cached
  `.md`, wired into the openrouter/ollama clients (both gain `--ocr`). Suite **268 passed, 3 skipped**.

## 🧭 Research findings that still steer the rest (plan §1–§5)
- **Three PDF classes → two paths** via a cheap font+char+image census; **curves and scans are the
  same OCR path** (`curves.pdf` extracted 0 chars; Tesseract `ces` recovered the text).
- **4th class "extracts but garbled"** (no `/ToUnicode` → `sondě→sondI`) — now guarded by the
  converter's decode-sanity check; the valid-but-wrong-diacritics case needs a dictionary/LM
  hit-rate and rides with the OCR follow-up.
- **Recommended tools (permissive-first):** pdfplumber (landed) / PERO-OCR + Tesseract `ces` (OCR
  path, pending) / pandoc·mammoth (DOCX alternatives; python-docx landed) / Docling·MinerU (complex
  layouts — evaluate in #22). License traps: Marker (GPL + model revenue cap), PyMuPDF (AGPL),
  MinerU (custom), **flexiconv (undeclared — blocker)**.
