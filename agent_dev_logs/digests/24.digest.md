  `api_util/{teitok_read,teitok_alto,flexiconv_convert,bbox_scale}.py`) copied byte-identical from

  `api_util/xml_to_md.py` (TEITOK/ALTO → Markdown for document-level input), and

  backends share the quality filter, context-window builder, and schema validation with `llm_utils.py`.
- **As-built design note:** `llm_run.py` stays the *local* entry point (`BACKEND ∈ {transformers,
  vllm}`, raises otherwise); OpenRouter/Ollama are **separate entry points** sharing
  `llm_client_shared.py` — deliberately, to keep remote-only users off the torch/GPU stack. This
  supersedes the plan's original "unified 4-way `BACKEND` dispatch in `llm_run.py`" idea (see the
  As-built architecture section in `plans/24.plan.md`).
- **Fixed post-bootstrap**: the initial `Dockerfile` was an untouched `nlp-enrich` copy (broken build

  repo's own entry. `README.md` (was the strategy doc committed verbatim) replaced with real usage docs.
- **`tests/`** — `test_llm_client_shared.py` + `test_xml_to_md.py` (the two genuinely-new modules; the
  shared-engine files are covered by `atrium-nlp-enrich`'s own suite and intentionally not re-tested).
- **`.github/workflows/`** — full 8-workflow suite + `dependabot.yml`; `docker-compose.yaml` /

## 🔧 Refinements (2026-07-03 review pass — the 4 new modules only)
Full correctness/cleanup review of `openrouter_client.py`, `ollama_client.py`, `llm_client_shared.py`,
`api_util/xml_to_md.py`. All fixes unit-tested (10 new tests → **47 total, green**; ruff clean).
1. **[correctness] `--attach-as-file` was dead code.** `_build_attachment_content()` + the flag were
   documented but never called — `run_document_level()` always inlined the text, so the file-attachment
   path (a specific #24 ask) did nothing. Now wired via an optional `user_content_builder`.
2. **[correctness] Ollama `ensure_model_pulled` over-matched tags** — treated `qwen2.5:14b` as
   satisfying `qwen2.5:7b` (skipping the pull, then running against a missing tag; `None` tag names
   could crash). Now requires the exact tag; a bare name is satisfied only by `<name>:latest`.
3. **[simplification] De-duplicated ~60 lines** across `build_system_prompt` /
   `build_document_system_prompt` into `_collect_vocab_terms` / `_render_vocab_prompt` /
   `_fit_vocab_prompt` — behaviour preserved, hand-sync burden halved.
4. **[cleanup] `OUTPUT_DIR` default inconsistency** (two different literal defaults for one key) resolved
   in both clients.
5. **[robustness] `run_line_level`** no longer silently drops a line with a blank/non-numeric
   `page_num`/`line_num` (was mislabelled as a filter-skip) — coerces to `0` instead.
